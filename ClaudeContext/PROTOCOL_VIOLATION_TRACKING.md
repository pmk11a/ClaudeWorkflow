## üìà VIOLATION ANALYSIS FRAMEWORK

### **Root Cause Analysis**
```markdown
For each violation, investigate:
1. **Immediate Trigger** - What prompted the violation
2. **Underlying Pressure** - What pressured to skip protocol
3. **Knowledge Gap** - Was protocol unclear or unknown
4. **Habit Pattern** - Is this a recurring behavioral pattern
5. **System Issue** - Is protocol impractical or flawed
```

### **Impact Assessment**
```
**Severity Levels**:
- **Critical**: Led to significant rework or broken functionality
- **High**: Caused delays or quality issues
- **Medium**: Created inefficiencies or confusion
- **Low**: Minor protocol deviation with minimal impact
```

### **Prevention Strategy Development**
```markdown
For each violation:
1. **Identify trigger point** where violation occurs
2. **Create reminder system** to catch violation before it happens
3. **Modify environment** to make protocol adherence easier
4. **Update protocols** if they're impractical
5. **Build new habits** to replace violation-causing behaviors
```

## üìö VIOLATION LEARNING LIBRARY

### **Common Violation Patterns**
```markdown
**Pattern 1**: "Urgent task, skip planning"
- Frequency: High
- Impact: Often leads to trial-error and rework
- Prevention: Always take 5 minutes for TodoWrite, regardless of urgency

**Pattern 2**: "Simple fix, skip documentation"
- Frequency: Medium
- Impact: Knowledge loss, hard to replicate fixes
- Prevention: Use quick documentation templates for simple fixes

**Pattern 3**: "Almost done, skip verification"
- Frequency: Medium
- Impact: Issues discovered later, user dissatisfaction
- Prevention: Build verification into completion definition
```
